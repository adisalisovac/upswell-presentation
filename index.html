🤖

Every command runs on Claude

Each slash command (e.g. /upswell-specify) is a structured prompt that feeds Claude the right context — code, specs, standards — and gets a consistent, traceable output.

📄

AI drafts. Humans decide.

Claude generates the first version of every document — business rules, specs, plans, tasks, bug reports. Nothing is used until a human reviews and explicitly approves it.

🔒

Context is always scoped

Claude only sees what's relevant to the current step — the right spec, the right plan, the right task. This keeps outputs focused and prevents AI from drifting out of scope.

The result: Claude handles the volume and consistency — reading thousands of lines of legacy code, generating structured docs, enforcing rule IDs. The team handles judgment — deciding what to keep, approving each gate, catching what AI misses.

1 Analyze

Emina and Kristijan read the MudTrax codebase and extract business rules and architecture. Each item is tagged Keep / Skip / Merge before anything moves forward.

2 Specify

Emina describes the feature in plain language. The tool generates a formal spec with acceptance criteria and field definitions. Emina reviews and approves before anything else starts.

3 Plan

Kristijan turns the approved spec into a technical plan — component structure, API endpoints, database changes, and team dependencies. He approves it before dev starts.

🎨 Designer
4 Design

The designer analyses the spec, identifies what needs to change visually, and runs design discovery for complex features. Screens and component concepts are built directly in code on the prototype-design branch, presented and iterated with the UpSwell client until approved. Dev then reviews and implements from that branch.

5 Implement

Each developer generates their own task list from the plan and implements one task at a time. The tool is scope-guarded — it won't touch files outside the current task.

6 Legacy Check

Kristijan runs a post-build check comparing the implementation against every acceptance criterion. If more than 30% are missing, the feature is automatically escalated before QA begins.

7 QA + Regression

QA tests each acceptance criterion on the QA environment, files structured bug reports tied to spec rule IDs, and generates automated regression tests for full traceability.

Gates are enforced, not advisory. You cannot plan without an approved spec. You cannot implement without an approved task list and a signed-off design. The tool stops and tells you what's missing.
